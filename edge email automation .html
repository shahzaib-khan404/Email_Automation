Section 01

Project Overview

Purpose

Edge Edu receives a continuous volume of client support emails related to their online certification program. The goal of this project is to build a fully automated Gmail monitoring system that uses Claude AI (Anthropic) to generate formal, accurate, FAQ-based email responses — with a human approval step before any email is sent.

🎯
Primary Objective

Reduce manual email response time while maintaining response quality and accuracy by grounding every reply in the official Edge Edu FAQ knowledge base.

Goals

Out of Scope (v1.0)

The following are explicitly excluded from this version: web-based UI dashboard, multi-inbox support, auto-send without approval, CRM integration, analytics/reporting dashboard, and mobile app.

Component Breakdown

📬 Gmail Watcher

  • Polls inbox every 30 seconds
  • Filters unread messages only
  • Extracts sender, subject, body
  • Uses Google OAuth2 authentication

🤖 AI Response Engine

  • Calls Claude Sonnet via Anthropic API
  • Injects FAQ as system context
  • Generates formal corporate reply
  • Handles unknown questions gracefully

✅ Approval Workflow

  • Displays original email + AI draft
  • 3 actions: Approve / Edit / Skip
  • Only sends after explicit approval
  • Logs all decisions locally

📁 State Manager

  • Tracks all processed email IDs
  • Persists state across restarts
  • Marks emails as read after handling
  • Prevents duplicate processing

Section 04

AI Prompt Specification

The following is the exact system prompt that must be passed to the Claude API for every request. It defines the AI's identity, behavior rules, tone, and knowledge source.

SYSTEM PROMPT — Pass to Claude API as system parameter
ROLE:
You are a professional customer support representative for Edge Edu,
an online certification platform. Your job is to respond to
client emails in a formal, corporate tone.

KNOWLEDGE BASE:
Use ONLY the following FAQ content to answer questions.
Do NOT fabricate information. Do NOT guess.

{FAQ_CONTENT}  ← Full FAQ text injected here at runtime

BEHAVIOR RULES:
1. Always respond in formal, professional English.
2. Answer questions directly using FAQ content when possible.
3. If the topic is NOT in the FAQ, provide a helpful general
   answer — do not say "I don't know."
4. If you truly cannot answer, politely refer the client to:
   taldev@onedge.co
5. Never use casual language, slang, or emojis.
6. Do not use excessive bullet points — prefer clear paragraphs.
7. Sign every email as: Edge Edu Support Team
8. Begin with a professional greeting using the sender's name
   if available.
9. Do not fabricate program details, dates, or policies.
10. Max response length: 300 words unless more is needed for
    accuracy.

OUTPUT FORMAT:
- Email body only (no subject line)
- Start with: "Dear [Name]," or "Dear Learner,"
- End with: "Warm regards, / Edge Edu Support Team"

User Prompt Template

The following is the user-turn message format sent alongside the system prompt for each email:

// User message constructed at runtime for each incoming email
user_message = f"""

Section 06

Non-Functional Requirements

🔒 Security

  • API key via environment variable only — never hardcoded
  • token.json must be excluded from version control (.gitignore)
  • credentials.json must be kept private
  • No email content stored beyond processed IDs

⚡ Performance

  • AI response generated in under 10 seconds
  • Polling interval configurable (default: 30s)
  • Script must remain running without memory leaks
  • Graceful handling of API timeouts

🛡️ Reliability

  • Retry logic on Gmail API failures
  • Script must recover from network interruptions
  • Processed IDs persisted immediately after decision
  • Clear error messages with actionable guidance

🔧 Maintainability

  • FAQ content in a separate constants file
  • Configuration section at top of script
  • Clear inline comments throughout
  • README with full setup instructions
